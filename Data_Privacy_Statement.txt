DATA PRIVACY & SECURITY STATEMENT: ASR PIPELINE
===============================================
1. Local-First Processing Architecture
This ASR Pipeline is designed with a "local-first" architecture to ensure the security of sensitive research data. Unlike cloud-based speech-to-text services, all core processing—including audio standardization, transcription, and linguistic annotation—is performed on the researcher's local hardware.
Transcription: The pipeline utilizes OpenAI Whisper for high-fidelity speech-to-text. All inference is conducted locally using your system's CPU or GPU.
Alignment: The Montreal Forced Aligner (MFA) and the custom C-accelerated engines process data entirely within your local environment.
Linguistic Tagging: Grammatical and part-of-speech tagging via spaCy occurs locally without transmitting text to external servers.

2. Management of Personally Identifiable Information (PII)
Researchers should be aware that certain files generated by this pipeline contain biometric data that qualifies as PII:
Speaker Fingerprinting: The pipeline uses ECAPA-TDNN (via pyannote.audio) for vocal fingerprinting to identify and distinguish between participants.
Speaker Embeddings: These "vocal fingerprints" are stored as numerical embeddings in the ./models/embeddings/ directory.
Storage Warning: While these embeddings are processed locally, the resulting files are biometric identifiers. They should be handled, stored, and shared with the same level of security and encryption as the original raw audio files to comply with IRB and data protection protocols.

3. External Connectivity and Authentication
While the pipeline is designed for local processing, limited external connectivity is required for setup:
* Hugging Face Integration: The pipeline requires a Hugging Face User Access Token to download specific pre-trained models for diarization and enhancement.
* Data Transmission: This connection is used only for model authentication and downloading weights. No audio data or transcripts are uploaded to Hugging Face or any other third-party server during the execution of the pipeline.

4. Data Lifecycle and Intermediate Files
The pipeline generates intermediate files that researchers must manage to ensure participant privacy:
* _preprocessed directory: Standardized audio files created during Phase 1.
* diarization_data directory: JSON files containing speaker timing and identification data.
* logs directory: Technical logs that may contain filenames or basic metadata about the processed files.

5. Researcher Responsibility
This software is a tool for local processing. It is the responsibility of the researcher to ensure that the hardware used for processing is secure and that all output files (transcripts, TextGrids, and speaker embeddings) are stored in accordance with their institution's data management plan and applicable privacy laws (e.g., GDPR, HIPAA).
